# Search Index Schema Documentation

## Overview

The search index is a JSON file (`.design/.search-index.json`) that enables fast component and token lookup.

## Top-Level Fields

### version
- Type: String
- Example: "1.0.0"
- Purpose: Schema version for compatibility

### generated
- Type: ISO 8601 timestamp
- Example: "2025-11-20T00:00:00.000Z"
- Purpose: Track when index was last built

### framework
- Type: String
- Values: react, vue, angular, svelte, react-native, flutter, swiftui, jetpack-compose
- Purpose: Primary framework for this project

### components
- Type: Object
- Keys: Component names
- Values: ComponentEntry objects
- Purpose: Searchable component index

### tokens
- Type: Object
- Keys: Token names
- Values: TokenEntry objects
- Purpose: Searchable token index

### categories
- Type: Object
- Purpose: Components grouped by functional category

### statistics
- Type: Object
- Purpose: Index metrics and counts

## ComponentEntry Schema

Required fields:
- **name**: Component name (matches key)
- **path**: Relative path from .design/
- **category**: One of 6 categories
- **searchTerms**: Array of searchable keywords

Optional fields:
- **description**: Component purpose
- **tags**: User-defined tags
- **props**: Component props schema
- **hasStory**: Storybook story exists
- **hasTests**: Test file exists
- **figmaNodeId**: Figma node reference
- **figmaUrl**: Figma design file URL
- **score**: Relevance score (0-100)

## TokenEntry Schema

Required fields:
- **name**: Token name
- **value**: Token value (string/number/object)
- **type**: Token type (color, typography, spacing, effect, borderRadius)
- **searchTerms**: Searchable keywords

Optional fields:
- **path**: Source file path
- **usedInComponents**: Components using this token

## Categories

Six standard categories:
1. **layout** - Container, Grid, Stack, Flex, etc.
2. **navigation** - Link, NavBar, Breadcrumb, Menu, etc.
3. **form** - Button, Input, Checkbox, Select, etc.
4. **data-display** - Table, List, Card, Badge, etc.
5. **feedback** - Alert, Toast, Spinner, Progress, etc.
6. **overlay** - Modal, Popover, Tooltip, Drawer, etc.

## Search Terms

Search terms are automatically generated from:
- Component/token name (split on camelCase)
- Description words
- Tags
- Category name
- Prop names

Example for Button:
```json
"searchTerms": ["button", "click", "cta", "call", "action", "submit", "form"]
```

## Scoring Algorithm

Components are scored 0-100 based on:
- Exact name match: +50 points
- Search term match: +10 points per term
- Has Storybook story: +10 points
- Has tests: +10 points
- Recent usage: +10 points
- Category match: +10 points

## Usage

Search index is:
1. Generated by `build-search-index.js`
2. Read by `search-components.js`
3. Updated on each transformation
4. Cached for performance

## Example Search Queries

### Find buttons
```
"button" → Returns all button components
```

### Find form components
```
"form" → Returns all components in form category
```

### Natural language
```
"input for email" → Matches Input component with high score
"primary color token" → Matches primary color token
```

## Index Location

Default: `.design/.search-index.json`

Can be customized in `.design/config.json`:
```json
{
  "search": {
    "indexPath": ".design/.search-index.json",
    "autoRebuild": true
  }
}
```

## Maintenance

Index should be rebuilt when:
- New components generated
- Components modified
- Tokens updated
- Manual rebuild requested

Rebuild command:
```bash
node .claude/scripts/build-search-index.js
```
