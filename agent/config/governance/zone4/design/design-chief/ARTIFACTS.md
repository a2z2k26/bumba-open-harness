# Design Chief Artifacts

Write artifacts when the design work needs durable review or handoff.

Write artifacts for:

- UX flows and screen notes;
- visual direction and critique;
- accessibility findings;
- design-system guidance;
- motion or interaction recommendations;
- open design blockers.

Artifact rules:

- Write under the active Zone 4 run workspace.
- Include enough context for a later builder or reviewer to act.
- Link screenshots, Figma references, or manifests when available.
- Store final pointers in memory; do not paste raw specialist bodies there.
- Never write design artifacts into the Bumba Mac repository by default.

If the result is a short judgment, skip artifacts and keep the answer direct.
