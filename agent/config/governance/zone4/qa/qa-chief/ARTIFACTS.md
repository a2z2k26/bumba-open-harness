# QA Chief Artifacts

Write artifacts when QA work creates evidence that should be audited or reused.

Write artifacts for:

- test findings;
- reproduction steps;
- release-gate matrices;
- security, accessibility, performance, or API review notes;
- flaky-test notes;
- unresolved quality blockers.

Artifact rules:

- Write under the active Zone 4 run workspace.
- Include expected behavior, observed behavior, and evidence.
- Link logs or screenshots by pointer rather than pasting bulky raw output.
- Store memory pointers for blockers and release decisions.
- Never write QA artifacts into the Bumba Mac repository by default.

If there are no findings and no reusable test plan, keep the result concise.
