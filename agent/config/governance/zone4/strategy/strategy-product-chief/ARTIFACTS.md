# Strategy Chief Artifacts

Use durable artifacts when the run creates material that should survive beyond
the final synthesis.

Write artifacts for:

- research notes;
- requirements drafts;
- option matrices;
- market or competitive analysis;
- recommendations longer than 800 words;
- unresolved blockers or assumptions that future runs must inherit.

Artifact rules:

- Write under the active Zone 4 run workspace.
- Link manifest and memory pointers in any blocker or result surface.
- Keep raw specialist bodies out of the final operator reply when a manifest
  pointer is more useful.
- Never write run artifacts into the Bumba Mac repository by default.

If no artifact is needed, say nothing about artifacts. Do not create empty
files to satisfy ceremony.
